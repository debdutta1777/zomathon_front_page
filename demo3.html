Step‑by‑Step Implementation

How we built the solution – from data to impact

🌿
🌶️
🌶️

1. Dataset Simulation

Generated 15,000 orders with realistic restaurant biases, time patterns, and other factors.

2. Data Cleaning

Removed invalid timestamps, converted types, dropped NaNs. Result: 12,519 clean orders.

3. Feature Engineering

Time features, cyclical encoding, rolling statistics (15min orders, historical avg). No leakage.

4. RLRI (Rush Index)

RLRI = 0.6*density_ratio + 0.4*noisy_inflation – captures kitchen congestion in real time.

5. Baseline Model

LightGBM predicts noisy KPT using basic features. Baseline P50 error: 4.35 min.

6. Error Correction

Second LightGBM predicts baseline error using RLRI, lag features, and base_kpt.

7. Dispatch Simulation

Rider sent at max(0, adjusted_KPT - travel_time). Wait time = max(0, true_ready - arrival).

8. Wait Reduction

Avg wait dropped from 6.28 min to 1.30 min (79.3% reduction). Most orders see 60–90% improvement.

9. Final Impact

P50 error ↓45.5%, P90 error ↓40.1%. Ready for A/B test – transformative for rider experience.
