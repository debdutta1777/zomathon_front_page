🚀 Our Solution

A two‑stage machine learning pipeline that turns noisy estimates into smarter dispatch, cutting rider wait time by 79%.

🍽️⏱️

The Problem

Restaurants provide biased, noisy kitchen preparation time (KPT) estimates. Riders dispatched based on these numbers face long waits or deliver cold food. Our solution corrects this in real time.

⚡

RLRI – Rush Level Risk Index

A novel real‑time metric that quantifies kitchen congestion using order density and noisy inflation.

RLRI = 0.6·density_ratio + 0.4·noisy_inflation
• Backward‑looking • No future data • Live at prediction
🧠

Two‑Stage Modeling

First model mimics the biased estimator; second corrects its error using RLRI and lag features.

Base Model → Correction
• Robust • Interpretable • LightGBM
⏳

Strict Temporal Split

All features use only past data. Chronological split (80/20) mirrors real‑world deployment.

Train (80%) Test (20%)
